# Demo guiada — Aplomo Systems para Cooper/T. Smith

Última actualización: 2026-04-29

## Objetivo

Presentar Aplomo Systems como una consola visual para controlar patios industriales, materiales, GPS, evidencia y operaciones en campo.

La demo no debe venderse como una página web simple. Debe explicarse como un primer MVP funcional de una plataforma industrial local-first, pensada para crecer hacia SaaS, app móvil, backend público, base de datos cloud, analítica, sensores, drones e IA.

## Narrativa principal

Aplomo Systems convierte una operación física de patio en datos útiles: ubicación, zonas, materiales, registros, responsables, evidencia, estados y trazabilidad.

La primera versión está enfocada en Cooper/T. Smith, pero la arquitectura y la narrativa ya están pensadas para otras industrias físicas: puertos, cementeras, minería, construcción, terminales, patios ferroviarios, acereras, canteras, logística pesada e inmobiliarias industriales.

## Flujo recomendado de presentación

### 1. Abrir el sitio

Abrir:

https://aplomosystems.com

Mensaje clave:

"Este es el MVP web público de Aplomo Systems. No es una maqueta aislada; ya está publicado en internet y preparado para enseñarse como producto."

### 2. Pantalla inicial

Mostrar la intro y avanzar por los slides.

Mensaje clave:

"La entrada resume el propósito del sistema: patios industriales, operación en campo, GPS, evidencia y crecimiento futuro hacia cuentas y app móvil."

### 3. Entrar a la consola

Presionar "Entrar al sistema".

Mensaje clave:

"Esta es la consola operativa. Desde aquí un supervisor puede visualizar zonas, materiales, registros, estado de capturas y operación del patio."

### 4. Mapa operativo

Mostrar el mapa.

Mensaje clave:

"El mapa permite llevar la operación física a una vista digital. La meta es que las pilas, zonas, perímetros y capturas puedan registrarse visualmente."

### 5. GPS del dispositivo

Probar permiso de ubicación si el dispositivo lo permite.

Mensaje clave:

"En celular, el navegador puede usar GPS, Wi-Fi y red móvil. En laptop la precisión puede ser menor, pero para demo permite validar el flujo."

### 6. Punto o perímetro

Mostrar captura de punto o dibujo de perímetro.

Mensaje clave:

"Esto permite registrar una ubicación o delimitar una zona de material en campo."

### 7. Paquete GPS / auditoría

Mostrar paquete, checklist o estado de captura.

Mensaje clave:

"No solo se guarda un punto. Se busca guardar contexto: quién capturó, cuándo, con qué precisión, qué material o zona representa y en qué estado está."

### 8. Simulación campo-supervisor

Mostrar la idea de celular capturador y supervisor receptor.

Mensaje clave:

"El flujo final es que alguien capture en campo y el supervisor lo reciba, revise y use para tomar decisiones."

### 9. Local-first

Mensaje clave:

"El sistema está pensado para operación real: si no hay internet estable en campo, la captura debe poder guardarse localmente y sincronizarse después."

### 10. Siguiente fase

Explicar sin prometer de más:

- Autenticación real.
- Empresas, usuarios, roles y permisos.
- Backend público.
- Base de datos PostgreSQL.
- Evidencia con fotos y documentos.
- Sincronización real.
- App móvil o PWA.
- Integraciones con drones, sensores y analítica.

## Qué sí se puede afirmar

- Existe un MVP web público.
- El dominio productivo ya funciona.
- Hay una consola visual operativa.
- Hay módulos de GPS, mapa, registros, materiales, evidencia y simulación.
- Hay arquitectura monorepo y base técnica modular.
- Hay frontend desplegado.
- Hay backend GPS local como base técnica.
- El producto está diseñado para crecer como SaaS industrial.

## Qué no conviene prometer todavía

- No decir que ya existe backend público completo si no está desplegado.
- No decir que ya hay autenticación empresarial real.
- No decir que ya existe app móvil nativa.
- No decir que ya se integra con drones o sensores reales.
- No decir que ya tiene IA productiva.
- No venderlo como sistema terminado; venderlo como MVP serio con ruta clara.

## Frase corta para cerrar

"Aplomo Systems busca que Cooper/T. Smith pueda convertir su patio físico en una operación digital trazable: mapas, GPS, materiales, evidencia, registros y decisiones."
